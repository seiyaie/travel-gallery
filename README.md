# Webサイト：Travel Gallery

## 概要

過去に訪れた国々で撮影した写真をまとめたギャラリーサイトです。  
情報設計・デザイン・コーディングをすべて自身で行いました。

単に写真を並べるのではなく、閲覧者が写真の世界観に自然と入り込める体験を目的とし、アニメーションやレイアウトを含めて全体を設計しています。

## 設計

本サイトでは「写真に集中できる体験」を軸に構成を設計しました。

まずビジュアルによって世界観を提示し、その後に必要な情報が自然に理解できるよう、コンテンツの配置と閲覧の流れを整理しています。

また、各国ごとのページはレイアウトや操作を共通化し、表示内容のみを切り替える構成としました。  
これにより、操作に迷うことなく写真鑑賞に集中できる体験の一貫性を担保しています。

さらに、デバイスによる体験差をなくすため、hoverに依存しないインタラクション設計とし、スマートフォンやタブレットでも同様の体験が得られるようにしました。

---

## 工夫した点

- **操作を遮らないアニメーション設計**  
  ユーザーの操作を妨げないよう、流れを意識したアニメーションを設計。画面遷移や表示切り替えにも動きを持たせ、自然な体験を実現

- **直感的に伝わる操作性への配慮**  
  スクロールトリガーを活用し、クリック可能な要素が視覚的に分かるように設計。デバイスに依存しない操作性を意識

- **国ごとの世界観を表現する背景演出**  
  背景画像を国ごとに切り替え、スクロールに応じて暗くなる演出を追加。視線を自然にコンテンツへ誘導

- **写真に集中させるギャラリーレイアウト**  
  1行1枚構成と左右交互レイアウトにより、リズム感と視線誘導を両立

- **体験と連動するハンバーガーメニュー**  
  スクロール位置に応じて背景が切り替わる仕組みを導入し、ナビゲーション自体も体験の一部として設計

- **ライトボックスの演出設計**  
  clip-pathとGSAPを組み合わせたアニメーションを実装し、単なる拡大ではなく体験として成立させる工夫

## 技術選定の理由

アニメーションにはGSAPを採用し、ScrollTriggerやタイムラインによる複雑な制御を実現しました。

また、JavaScriptはフレームワークを使用せず、素の構文で実装することで挙動の理解と保守性を重視しています。

## 学び

本制作を通して、アニメーションは単なる装飾ではなく、体験の流れを整え、操作を補助する重要な要素であると理解しました。

また、写真を主役とするコンテンツでは、レイアウトや余白、背景の設計が視線誘導や没入感に大きく影響することを実感しました。

技術面では、GSAPによるタイムライン設計やScrollTriggerの活用を通して、アニメーションを構造的に設計する考え方を身につけました。
